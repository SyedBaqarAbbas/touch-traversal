# ADR-0001: Use an authenticated loopback Python companion for personal ingestion

- Status: accepted
- Date: 2026-07-18
- Decision owner: THO-63

## Context

The public application is a static Next.js export. Its sample graph is generated by the Python
pipeline and loaded from four checked-in JSON files. Personal ingestion must preserve that pipeline's
stable identifiers, explainable edges, four layouts, and Pydantic/Zod artifact contract without
turning note contents into a hosted upload.

## Decision

Personal graph generation runs in an explicit localhost-only Python companion. The static browser
application remains a sample viewer when the companion is absent. When a capability probe succeeds,
the browser may send selected note contents over HTTP to `127.0.0.1`; the companion materializes
them in a temporary directory, runs the existing pipeline, validates all four artifacts, and returns
one in-memory bundle. The browser validates the bundle again before constructing the existing
Graphology model.

The initial implementation is a full-pipeline provider, not a hosted service and not a second graph
algorithm. Launch it with:

```bash
cd pipeline
uv sync --extra embeddings --extra layouts --all-groups
uv run touch-traversal studio
```

Custom static origins must be named explicitly with `--allow-origin https://example.test`.

## Alternatives considered

| Criterion                   | Browser-native worker                                                             | Localhost Python companion (selected)                                                                   | Hybrid browser + companion                                        |
| --------------------------- | --------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| Note privacy                | Contents remain in one browser process.                                           | Contents cross loopback only and never reach a hosted note endpoint.                                    | Depends on mode; harder to explain and audit.                     |
| Pipeline fidelity           | Requires a TypeScript/WASM port of ingestion, relations, clustering, and layouts. | Reuses the current deterministic Python implementation and Pydantic models.                             | Basic and enhanced graphs can disagree.                           |
| Schema reuse                | Zod is native, but generation logic and fixtures are duplicated.                  | Python produces the established schema; Zod revalidates the returned bundle.                            | Both generators must be versioned and compared.                   |
| First-run model acquisition | Browser must download and cache model assets within browser limits.               | `uv` installs extras; Sentence Transformers can acquire model weights locally on first use.             | Users face both browser and Python acquisition paths.             |
| CPU and memory              | Competes with WebGL and browser memory limits.                                    | Pipeline work is isolated in a local process, but still uses the user's CPU and RAM.                    | Splits pressure but duplicates intermediate data.                 |
| Cancellation                | Worker termination is direct.                                                     | Cooperative cancellation occurs at deterministic stages; an active model call completes before cleanup. | Two cancellation mechanisms and partial states.                   |
| Browser support             | Strong static-host fit, subject to worker/model APIs.                             | Requires loopback fetch, CORS, and browser Private Network Access support.                              | Browser support varies by selected mode.                          |
| Packaging                   | No companion install, but large worker/model packaging.                           | Current `uv` command is explicit; a signed desktop package can wrap it later.                           | Must package and maintain both paths.                             |
| Static hosting              | Personal mode works directly on the static origin.                                | Static site is sample-only until the local capability probe succeeds.                                   | Basic mode works statically; enhanced mode still needs localhost. |
| Maintenance                 | Reimplements the most complex subsystem.                                          | One graph-generation implementation and one versioned transport boundary.                               | Highest ongoing contract and test cost.                           |

The worker option is attractive for frictionless static hosting, but a faithful port is a separate
product-sized effort. The hybrid option is rejected because two graph meanings would undermine
determinism and make privacy/capability copy conditional. This decision can be revisited if the
pipeline is intentionally compiled to a browser runtime rather than manually reimplemented.

## Provider boundaries

1. **Note intake** creates validated `{name, relativePath, mediaType, content}` values in browser
   memory after explicit file/folder selection and metadata-only preview.
2. **Graph generation** is `LocalStudioProvider`, which accepts only an HTTP loopback endpoint.
3. **Bundle validation** uses the existing Python cross-artifact validator before response and Zod's
   `artifactBundleSchema` after response.
4. **Session storage** is an explicit in-memory store. Activation validates and builds the existing
   Graphology model; reset drops the bundle and model references.
5. **Scene loading** receives that already-validated `GraphModel`. Sample loading from
   `apps/web/public/data/` is unchanged.

## Security and transport boundary

- The server rejects non-loopback bind addresses before opening a socket.
- Browser origins must be present in the explicit process allowlist; an arbitrary loopback port is
  not trusted automatically. Disallowed origins receive no CORS grant.
- `GET /v1/capabilities` negotiates contract version 1 and returns a random per-process token.
  Every job, result, cancel, and cleanup request requires that token as a bearer credential.
- Private Network Access preflight receives `Access-Control-Allow-Private-Network: true` only after
  origin validation.
- The HTTP handler suppresses request logging and server error tracebacks. Progress and failures do
  not contain note bodies.
- The browser provider rejects non-loopback URLs before performing `fetch`, so a configuration typo
  cannot redirect a note-bearing request to a remote host.

This protects the intended local workflow; it is not a sandbox against malicious code already
running on an allowed static origin or as the same operating-system user. A local process can try
to impersonate the companion on port 8765, and an allowed origin receives the process token from
the capability response.

## Versioned job contract

Contract version 1 exposes `accepted`, `materializing`, `ingesting`, `chunking`, `relating`,
`embedding`, `laying_out`, `validating`, and `complete` in that order. Requests, snapshots, results,
and typed failures carry `contractVersion: 1`, a caller request ID, and a companion job ID.

Cancellation uses `DELETE /v1/jobs/{jobId}`. It is cooperative between pipeline stages; the
temporary workspace is removed on success, typed failure, or cancellation. A second delete after a
terminal state removes its in-memory result. Terminal jobs also expire automatically after five
minutes; the process runs at most two builds and retains at most eight job records. See
[Personal ingestion](personal-ingestion.md) for the endpoint and startup contract.

## Personal-data lifecycle

| Location                      | Personal data                                       | Persistence and cleanup                                                                                         |
| ----------------------------- | --------------------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| Browser intake memory         | Selected filenames and contents                     | Retained after success or cancellation while Studio remains mounted; released by clear/reset or page lifecycle. |
| Loopback request              | JSON note contents                                  | Request body is not logged or written as an HTTP payload.                                                       |
| Companion temporary directory | Materialized notes and note-derived embedding cache | Unique per job; removed on success, failure, or cancellation.                                                   |
| Model cache                   | Downloaded model weights                            | May persist in tool-managed caches; it contains no note text or note-derived vector.                            |
| Companion job memory          | Validated bundle, including full chunk text         | Held until authenticated cleanup, five-minute expiry, queue pruning, or process exit.                           |
| Browser session memory        | Validated bundle and Graphology model               | Removed by session reset or page lifecycle.                                                                     |
| Browser `sessionStorage`      | Traversal history identifiers                       | Reset attempts removal; if storage blocks it, close the tab or clear site data.                                 |
| IndexedDB and localStorage    | Nothing in version 1                                | Personal notes and bundles are never written there.                                                             |
| Explicit export               | Serialized validated bundle, including chunk text   | Created and downloaded only after **export private JSON** is activated.                                         |
| Tracked public data           | Nothing                                             | Studio builds have no output-path parameter and never write `apps/web/public/data/`.                            |

## Consequences

- The public demo, mouse and keyboard input, reduced motion, and optional camera behavior are
  untouched.
- Personal mode is truthfully unavailable when the probe fails; recovery copy supplies the exact
  local launch command.
- Users install a local Python toolchain and optional model/layout dependencies. Pipeline failures
  remain typed and actionable.
- Some browsers or managed environments may block loopback Private Network Access. Version 1 does
  not silently fall back to remote ingestion or a lower-fidelity browser graph.
